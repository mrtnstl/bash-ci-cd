# bash-ci-cd

Continuous Integration and Continuous Delivery runner in bash.

![demo](https://github.com/mrtnstl/bash-ci-cd/blob/main/docs/bash_demo.gif "demo")

The pipeline steps are defined in the `tasks` directory. These could be easily extended if needed.

## What it does?

the pipeline does
1. checks the code out from a git repository
2. installs dependencies
3. runs linter
4. runs tests
5. builds the software
6. [not implemented] builds docker image
7. [not implemented] pushes docker image to artifact repository
8. [not implemented] spins up the new instance of the application on the server
9. [not implemented] sends email notification when it's done or an error occurs

## Run

1. Set your variables in `config.sh`. Create it in the project root, if it doesn't exist.

    **WARNING: these values are sitting in your config file in plain text!**

    config.sh
    ```bash
    #!/usr/bin/env bash

    # currently used for testing
    ENV="production"
    # project repo url
    REPO_URL="https://github.com/your-user/your-project.git"
    # branch name
    BRANCH="main"
    # the directory where the project will be cloned, tested, built, etc.
    WORKDIR="/tmp/some-folder-name"
    # directory for logs
    LOGS_DIR="${WORKDIR}/logs"
    # directory for artifacts, like test coverage reports
    ARTIFACTS_DIR="${WORKDIR}/artifacts"
    # temp directory, currently helps with workflow log file naming
    TEMP_DIR="${WORKDIR}/temp"
    ```

2. Make sure you have execution privileges for `start.sh`. The script will attempt to set it to the rest of the files where needed.

    ```bash
    chmod u+x ./start.sh
    ```

3. Run script

    ```bash
    ./start.sh
    ```

## Artifacts

Files generated by the pipeline, should be placed in `$ARTIFACTS_DIR`.

For example the coverage report from testing will be at `$ARTIFACTS_DIR/test-coverage`.

These files are automatically removed before each workflow run!

## Logs

Log files are collected in `$LOGS_DIR`.

These files will not be deleted by the workflow.
